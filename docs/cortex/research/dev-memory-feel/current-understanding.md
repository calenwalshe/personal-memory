## Durable Findings

*(Inherited from memory-phenomenology dossier — verified by concept research)*

- Proactive surfacing (unprompted) is the mechanism of felt continuity — passive `/recall` produces information access, not recognition. Source: docs/cortex/research/memory-phenomenology/concept-20260414T130000Z.md
- Precision over recall: confident wrong memory damages trust more than a gap. Source: ibid.
- Tulving's episodic/semantic distinction applies: facts without temporal/contextual anchoring feel like data retrieval, not being known. Source: ibid.

## Provisional Thoughts

- **[PROVISIONAL]** The most impactful developer memory moments are project-specific context, not generic preferences — "you've been working on the auth refactor for 3 sessions" > "you prefer TypeScript".
- **[PROVISIONAL]** A minimal viable demonstration of "feeling known" requires only 5-10 sessions and one well-chosen proactive surfacing moment — the vault doesn't need to be deep to produce the felt experience.
