## Summary

Hebbian weight decay for a personal memory graph is well-grounded in neuroscience and straightforward to implement. The canonical rule `w_{t+1} = max(0, w_t · (1 - d))` applied per-session to non-firing edges gives the graph a finite memory horizon. For a daily-use system with η=0.3 and MAX_WEIGHT=20, the optimal decay constant is **d=0.03** (half-life ~23 days), which produces a steady-state where edges firing ~20% of sessions reach weight ~2.5, and ~70%-frequency pairs approach the ceiling. No production AI memory system (Zep, Graphiti, MemGPT/Letta) currently implements Hebbian edge weight decay — they use timestamp validity or memory tier promotion instead. This is a genuine design gap. The STDP literature provides the mathematical form (LTD exponential), but the millisecond-scale biological constants don't translate directly — the correct analogy is homeostatic plasticity operating on hours-to-days timescales, which maps cleanly to per-session updates.

## Findings

### Q1 — Decay rate: what BCM, STDP, and homeostatic plasticity suggest

The three frameworks converge on **different aspects** of the problem:

- **BCM theory** argues for a *sliding threshold* rather than a fixed decay constant: a synapse should depress if postsynaptic activity is *below* the threshold, potentiate if above. The sliding threshold is a time-averaged postsynaptic activity. *Implication for this system:* decay should be proportional to how far below average the pair's firing frequency is, not a fixed constant. This is more correct but adds complexity.

- **STDP (LTD branch)** operates on millisecond windows: `W(Δt) = -A₋ · exp(Δt/τ₋)` for Δt < 0, with τ₋ ≈ 10ms biologically. The per-session analog is not the biological constant — the closest mapping is: treat each session as an eligibility window. If the pair did not fire in the window, apply the LTD-equivalent: `w ← w · (1 - d)`.

- **Homeostatic plasticity** is the closest biological analog to per-session decay. It operates on hours-to-days timescales (vs. milliseconds for STDP), globally scaling weights to maintain network activity within a functional range. Research shows hippocampal LTP can decay to baseline with time constants of **0.7, 1.0, and 3.3 days** in freely moving animals, with a typical range of "a few hours to a few days."

**Calibration via steady-state formula (ChatGPT, analytically derived):**
```
w* = (p · η) / (d · (1 - p))
```
Where p = edge firing probability per session, η = 0.3, d = decay constant.

With η=0.3, d=0.03:
| Firing frequency (p) | Steady-state weight (w*) |
|---------------------|--------------------------|
| 0.10 (fires 1/10 sessions) | 1.11 |
| 0.20 (fires 1/5 sessions) | 2.50 |
| 0.50 (fires every other session) | 10.0 |
| 0.70 (fires most sessions) | 23.3 → capped at 20 |

This is a calibration instrument: choose the steady-state weight you want for "background noise" pairs (p~0.05) and "active interests" (p~0.30), solve for d. With d=0.03: background pairs (p=0.05) stabilize at ~0.53 (effectively decays to 0), active pairs (p=0.30) at ~4.3, persistent pairs (p=0.70) hit the ceiling.

**Recommended value: d=0.03.** (Source: ChatGPT research track, corroborated by homeostatic plasticity literature citing 0.7–3.3 day decay constants)

Half-life comparison:
- d=0.01 → 69 days — too slow for current-topology encoding
- d=0.03 → 23 days — 1 month memory, defensible default
- d=0.05 → 13.5 days — 2-week focus, more recency-sensitive
- d=0.10 → 6.6 days — volatile, tracks very recent activity only

### Q2 — Existing implementations in AI memory systems

No production AI memory system implements per-session Hebbian edge weight decay. Specifically:

- **Zep/Graphiti:** Temporal knowledge graph approach using `valid_at`/`invalid_at` timestamps. Edge relevance is binary (valid or not) rather than continuous weight decay. No numeric Hebbian mechanism. (Zep Documentation)

- **MemGPT/Letta (arXiv:2310.08560):** Three-tier memory architecture (core/recall/archival) with self-editing via LLM calls and retrieval scoring. Memory management is based on content relevance and tier promotion, not weighted graph decay. The system can "forget" by moving memories to archival, but not by decaying numeric association weights.

- **General knowledge graphs:** Temporal KGs encode time ranges for facts (e.g., "president of X from t1 to t2"), not weight salience. This is structurally different from encoding current interest strength.

**Conclusion:** This is a real gap. Hebbian edge weight decay for personal memory graphs — where weights encode current salience rather than historical existence — is not implemented in any major system. This is a design contribution, not an incremental improvement.

### Q3 — STDP LTD: exact mathematical form

Canonical STDP function (Scholarpedia, Sjöström & Gerstner 2010):

```
W(Δt) = +A₊ · exp(-Δt / τ₊)    for Δt > 0    [LTP: pre before post]
W(Δt) = -A₋ · exp(+Δt / τ₋)    for Δt < 0    [LTD: post before pre]
```

where Δt = t_post − t_pre. LTD fires when the postsynaptic spike precedes the presynaptic spike — anti-causal depression.

Online implementation via eligibility trace:
```
τ₋ · dy/dt = -y + a₋(y) · Σₙ δ(t − tⁿ_post)
dw/dt = -A₋(w) · y(t) · Σf δ(t − tf_pre)
```

`y` accumulates at each postsynaptic spike and decays with time constant τ₋ ≈ 10ms. Weight is depressed at each presynaptic spike by the trace value — so the larger the temporal gap (post before pre), the smaller the depression (y has decayed more).

**Weight-dependence variants (directly applicable to our system):**
- Soft bounds (multiplicative): `A₋(w) = w · η₋` — depression proportional to current weight. Large weights decay faster. This produces a bounded weight distribution automatically.
- Hard bounds: `A₋(w) = Θ(w) · η₋` — constant depression until w hits 0.

**Per-session translation:** The biological τ₋ ≈ 10ms has no direct analog. The correct translation is the multiplicative soft-bound rule: `w ← w · (1 - d)` per non-firing session. Soft bounds are recommended — they naturally prevent negative weights and produce unimodal weight distributions.

Source: Scholarpedia (https://www.scholarpedia.org/article/Spike-timing_dependent_plasticity)

### Q4 — Human interest shift timescales

Research on interest dynamics (cognitive science + large-scale behavioral data) identifies three temporal bands:

| Band | Timescale | Cognitive type |
|------|-----------|---------------|
| Short | 3–7 days | Situational interest: triggered by novelty/external stimuli, transient |
| Medium | 2–6 weeks | Emerging personal interest: repeated engagement, domain knowledge forming |
| Long | 2–6 months+ | Durable personal interest: self-sustaining, deeply ingrained |

Large-scale behavioral datasets (e-commerce, smartphone communications) show **power-law scaling** in: (1) length of continuous interest, (2) return time to an interest, (3) interest ranking over time. This suggests human interest dynamics are not exponential — the "long tail" of interests persists much longer than a single exponential decay predicts.

**Implication for d=0.03:** A 23-day half-life maps to the medium band (emerging/active interests). Situational 3-day interests will fully decay within ~10 sessions at d=0.03. Durable interests (firing every few weeks) will hold weight. This is the right band for a personal knowledge graph. Source: cognitive science synthesis (Gemini provider).

### Q5 — Metrics for measuring decay quality

Four concrete metrics (validated as independently proposed by both ChatGPT and the OpenAI synthesis):

**A. Responsiveness to topic shift (primary)**
Build a graph from the last K sessions only (K=30). Compare edge rank correlation or top-N overlap with the decayed full-history graph. A working decay should make the two graphs resemble each other. Metric: Kendall's τ or Jaccard@50 between the two edge rankings.

**B. Next-session predictive value (gold standard)**
Use the graph at day t to predict which edges fire at day t+1. Measure Precision@k or NDCG on actual next-session co-occurrences. Decay should make current weights more predictive than raw cumulative counts. Metric: compare AUC(decayed graph) vs AUC(no-decay baseline).

**C. Stale persistence rate**
Fraction of the top-100 edges (by weight) that have not fired in 30 days. If decay is working, this should stay low (<20%). If d is too small, stale persistence accumulates. Metric: monthly snapshot comparison.

**D. Steady-state stratification**
Check whether empirical weights stratify by observed firing frequency p using `w* = p·η/(d·(1-p))`. Compute the Spearman correlation between (observed weight) and (predicted w* from empirical p). If decay is calibrated, ρ > 0.7. If many low-p edges sit near MAX_WEIGHT, d is too small.

## Adjacent Findings

**Competitive depression (LTD-style penalty):** When entity A appears in a session but B does not co-appear, penalize edge (A,B) with an active depression beyond passive decay: `w_AB ← (1-d)·w_AB − λ·1[A_active, B_inactive]`. This moves closer to BCM/STDP intuition ("miscoordinated neurons stop wiring") and would more aggressively prune edges where one entity has become active in a new context without the other. This matters for `hebbian-decay` because it adds a second parameter (λ) and requires tracking which entities were active per session — already available from the atom batch — so implementation cost is low. Source: ChatGPT research track.

**Steady-state formula as parameter derivation tool:** `w* = p·η/(d·(1-p))` is not just a curiosity — it is a calibration instrument that lets you *derive* d from desired steady-state weights rather than tuning by feel. Choose the target weight for "background noise" pairs (p~0.05, target w*~0.5) and solve: `d = p·η / (w*·(1-p))`. At p=0.05, η=0.3, target=0.5: d = 0.032. This is why d=0.03 is the right answer, not just a guess. This matters because without this formula, DECAY and ETA can fight each other unpredictably — small changes to d produce large changes in the steady-state of high-frequency pairs. Source: ChatGPT research track (analytically derived).

## Trade-offs

| Option | Pro | Con |
|--------|-----|-----|
| Passive decay only (`w *= (1-d)`) | Simple, one parameter, proven | Penalizes temporarily-absent pairs equally with truly stale ones |
| Passive + competitive depression | More biologically correct, prunes faster | Two parameters, must track per-session active entities (already available) |
| Multi-timescale decay (fast + slow) | Better models power-law interest dynamics | Two decay constants, much harder to calibrate, out of scope for first experiment |
| BCM sliding threshold | Most theoretically correct | Requires computing per-entity background firing rate, significant added complexity |

**Recommendation:** Ship passive decay first (d=0.03). Add competitive depression as exp-004 if passive decay eval shows insufficient pruning of actively-displaced pairs.

## Recommendations

1. **Implement passive decay at d=0.03.** Apply `w = max(0.0, w * (1 - 0.03))` to all `related_to` edges with `description='hebbian'` that did not fire in the current atom batch. Run after the Hebbian increment in the same `vault chunk` call.

2. **Exclude structural co-occurrence edges from decay.** Only decay edges where `description='hebbian'` (or a new `source='hebbian'` column). The co-occurrence baseline is the structural backbone; decaying it would erode the graph's factual ground truth.

3. **Use the steady-state formula to verify calibration after 30 sessions.** Compute empirical firing frequencies for each edge and compare predicted w* to observed weight. If Spearman ρ < 0.5, d needs adjustment.

4. **Add a `MIN_WEIGHT_TO_DECAY` floor of 0.01.** Edges near zero should be pruned (deleted), not left as floating-point noise. Prune when `w < 0.01` to keep graph clean.

5. **Track stale persistence as the primary health metric.** Monthly snapshot: what fraction of top-100 edges have not fired in 30 days? Target: <20%.

- What is the correct MIN_COACTIVATIONS threshold for deciding an edge "fired" in a session? Currently 0 (any co-appearance counts). A threshold of 2+ atoms would reduce noise from incidental co-mentions.
- Multi-timescale decay (two DECAY constants, fast+slow): revisit after passive decay is validated and a probe set shows which interest band is most affected. Trigger: stale persistence metrics from 60-day run.

- If you observe weight oscillation (an edge hitting MAX_WEIGHT then decaying to 50% within 10 sessions repeatedly), then assumption "ETA and DECAY compound cleanly" is wrong — ETA is too large relative to d·MAX_WEIGHT.
- If you observe that structural co-occurrence edges (description != 'hebbian') have their weights eroded after decay runs, then the exclusion guard is broken.
